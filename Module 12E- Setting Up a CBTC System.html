Module 12E: Setting Up a Computer Based Train Control System

Setting up a Computer Based Train Control (CBTC) system is a complex, multi-stage process that involves significant infrastructure upgrades, sophisticated technology deployment, and rigorous testing. The key phases and considerations include:

1. System Design and Planning

Define operational requirements, safety goals, architecture (centralized/distributed), communication needs, automation level, and integration with existing systems.

2. Wayside Equipment Installation

  • Install communication infrastructure (radio base stations, leaky feeders)
  • Deploy balises or transponders for positioning
  • Configure trackside interface units for switches and signals

3. Onboard Equipment Installation

  • Install odometers, balise readers, and other sensors
  • Add radio units for wireless communication
  • Set up onboard computers with ATP/ATO functions
  • Install driver-machine interfaces (DMI) if applicable

4. Central Control System Implementation

Set up the CBTC server, control software, monitoring displays, operator workstations, and backup systems for processing and train management.

5. Communication Network Setup

Ensure robust wireless coverage across the network for reliable, low-latency, two-way data exchange between trains and the central control center.

6. System Integration and Testing

All components must be tested together, both statically and dynamically. Testing includes normal operation, degraded modes, and performance validation under real-world conditions.

7. Staff Training

Train operators, maintainers, and dispatchers on the new technology. Training must cover normal and emergency procedures as well as troubleshooting techniques.

8. Phased Implementation and Cutover

Gradually roll out CBTC on existing lines using overlay strategies. Ensure smooth transitions without disrupting regular service. Final cutover must be coordinated and safety-certified.
