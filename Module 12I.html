Troubleshooting Computer Based Train Control

Troubleshooting a Computer Based Train Control system requires a systematic approach because CBTC depends on software, onboard equipment, wayside equipment, communication networks, central control systems, and train interfaces all working together.

When a CBTC problem occurs, maintainers must review alarms, system logs, communication status, train data, equipment status, and recent events to find the root cause.

1. Loss of Communication

A common CBTC problem is loss of communication between the train and the control system. When this happens, the train may enter a more restrictive fallback mode or stop, depending on the system design.

Troubleshooting should include checking radio base stations, access points, antennas, leaky feeder cables, onboard radio units, network switches, signal strength, interference, and communication logs.

2. Train Localization Errors

Train localization errors occur when the system does not accurately know where the train is located.

Possible causes include faulty odometers, wheel slip or slide, incorrect calibration, failed balise readers, damaged balises, antenna problems, or incorrect track database information.

Troubleshooting should include checking positioning logs, balise reader status, odometer readings, sensor calibration, and the condition of wayside reference points.

3. ATP Problems

Automatic Train Protection problems may include unwanted brake applications, speed enforcement problems, Movement Authority errors, or failure to properly supervise the train.

Troubleshooting ATP problems should include reviewing ATP event logs, train speed data, train position data, braking interface status, software configuration, and Movement Authority information.

4. ATO Problems

Automatic Train Operation problems may include rough acceleration, rough braking, incorrect stopping position, failure to follow the speed profile, or failure to start automatically.

Troubleshooting should include checking ATO logs, track profile data, stopping point data, onboard control commands, speed feedback, and train propulsion and braking interfaces.

5. Central Control System Problems

Central control system problems may include server failures, database errors, workstation problems, software faults, network failures, or loss of communication with wayside and onboard equipment.

Troubleshooting should include checking server health, control center alarms, network connectivity, system logs, database integrity, backup systems, and failover equipment.

6. Software and Configuration Issues

CBTC systems depend heavily on software and configuration data. A software problem or incorrect configuration can cause intermittent behavior, false alarms, incorrect limits, or system instability.

Troubleshooting should include reviewing software logs, error reports, recent software changes, configuration files, version control records, and vendor recommendations.

7. Interface Problems

CBTC must interface with many other railroad systems. These may include interlockings, switches, signals, platform screen doors, control center systems, and trainborne systems.

If an interface fails, the CBTC system may receive incorrect information or fail to send proper commands. Troubleshooting should verify inputs, outputs, relays, network messages, status indications, and interface logs.

8. Multidisciplinary Troubleshooting

CBTC troubleshooting often requires multiple departments working together. Signal maintainers, train control engineers, IT staff, communications technicians, vehicle maintenance, operations, and vendor support may all be needed.

This is because one CBTC problem may involve hardware, software, networking, train equipment, field equipment, and operating procedures at the same time.
